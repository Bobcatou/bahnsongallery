Below notes from Slack
@bobcatou: If you want it on all pages, you could add the code to an existing script that's already loaded on every page (maybe your navigation js file), that way you're not calling an extra script on every page.

To enqueue it, you can either look for the existing "wp_enqueue_scripts" function in your functions.php (most likely there's already one there) and just add your line inside that function:

```wp_enqueue_script(  'script-name',  get_stylesheet_directory_uri() . '/js/woo_height_align.js',  array(  'jquery'  ), '1.0.0'  );
```

or you can add a new function:

```add_action( 'wp_enqueue_scripts', 'customprefix_load_styles' );
function customprefix_load_styles {
    wp_enqueue_script( 'script-name',  get_stylesheet_directory_uri() . '/js/woo_height_align.js',  array( 'jquery' ), '1.0.0' );
}
```

I don't want to complicate thinks, but if you're going using this only for the shop page, then you can use a conditional to make sure it's only loading on that page...no reason to load it on all pages if you're not using it:

```if( is_shop() ) {
     wp_enqueue_script(  'script-name',  get_stylesheet_directory_uri() . '/js/woo_height_align.js',  array(  'jquery'  ), '1.0.0'  );
}
```

You may want to use it on many other page so you'll have to decide what's best, for example if you want to show featured products above the footer on every page, etc.

I imagine they don't add it to core WooCommerce because they want to keep the code to a minimum and avoid extra markup, js, etc. Even though you and I hate not having them the same height, not everyone cares so they probably decided it's best to leave it to the developer to decide if they want the extra code added.
